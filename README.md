# 🦊 혼자 하는 끝말잇기 (아이패드 웹게임)

7살 아이가 **혼자서** 끝말잇기를 이어가며 "이은 단어 수 + 점수"로 승부욕을 느끼게 만든 웹게임입니다.
키보드 대신 **마이크로 말하면 비슷한 단어 4개**가 나오고, 그중에서 골라 끝말을 계속 이어갑니다.

## 구성 파일
| 파일 | 설명 |
|---|---|
| `index.html` | 게임 본체 (HTML/CSS/JS 단일 파일) |
| `words.json` | 게임 단어 데이터 24,698개 (단어·레벨·뜻) |
| `serve.ps1` | 로컬 테스트용 서버 (PowerShell) |
| `build_dataset.ps1` | 원본 엑셀 → `words.json` 재생성 스크립트 |

## 게임 규칙
- 게임이 **시작 단어**(쉬운 단어)를 주면 그 **끝글자**로 시작하는 단어를 이어요. (두음법칙 적용: 예 …력 → 역/력)
- 마이크 🎤 를 누르고 말하면 **비슷한 단어 4개** 표시 → 골라요.
- 마이크가 어려우면 **📝 골라서 입력**으로 단어 목록에서 선택.
- **힌트 💡**: 1번(초성, 맞히면 1점) → 2번(뜻, 0점). 힌트 없이 맞히면 **2점**.
- **🤔 모르겠어요**: 가능한 단어 4개를 보여주고 골라서 계속(0점). 이 **이어가기 찬스는 4번**(⭐), 다 쓰면 종료.
- 레벨은 단어를 이을수록 **자동 상승**(5단어마다 1→5).
- 단어 옆 **🔊** 를 누르면 뜻을 읽어줘요.

## 1) 로컬에서 먼저 테스트 (PC)
PowerShell에서:
```powershell
./serve.ps1
```
브라우저에서 **http://localhost:8080** 접속.
> `localhost` 는 보안 컨텍스트라 크롬에서 **마이크 테스트도** 됩니다. (마이크 권한 허용)

## 2) 아이패드에서 쓰려면 — 온라인 호스팅 (HTTPS 필수)
음성인식은 **HTTPS + 인터넷**이 필요합니다. 무료 정적 호스팅 예시:

### GitHub Pages
1. GitHub 새 저장소 생성 → `index.html`, `words.json` 업로드.
2. 저장소 **Settings → Pages → Branch: main / root** 저장.
3. 몇 분 뒤 `https://<아이디>.github.io/<저장소>/` 주소가 생성됨.

### 그 외
- **Netlify / Vercel / Cloudflare Pages**: 폴더 드래그&드롭 또는 깃 연결로 즉시 HTTPS 배포.

## 3) ⚠️ 아이패드 사파리 주의사항 (중요)
- 반드시 **사파리(Safari)에서 직접 URL 접속**하세요.
  - "홈 화면에 추가"한 아이콘(PWA)이나 카카오톡 등 **인앱 브라우저에서는 마이크가 동작하지 않습니다.**
- 처음 마이크를 누르면 **"음성 인식 허용"** 모달이 떠요 → 허용.
- 인터넷 연결이 있어야 음성인식이 됩니다(오디오를 애플 서버로 보내 처리).
- 음성이 잘 안 되면 **📝 골라서 입력**으로도 똑같이 즐길 수 있어요.

## 데이터 다시 만들기 (선택)
원본 `국어 기초 어휘 선정 및 어휘 등급화 목록 전체.xlsx` 에서 재생성:
```powershell
# 한글 인코딩 위해 BOM 포함본을 만들어 실행
./build_dataset_bom.ps1
```
필터 기준: 명사만 / 2글자 이상 / 순수 한글 / 중복은 가장 쉬운 등급으로 / 두음법칙 기준 **끝말 연결이 안 되는 단어 제외**(반복 정제).
